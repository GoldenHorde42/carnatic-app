# Ideas — Feb 25, 2026

---

## 1. Smarter Quota Usage — Stop Re-Fetching Known Videos

**Problem:** Every `?seed=true` re-fetches the same 50 videos per artist, burning ~100 quota units even though most are already in the DB.

**Ideas:**
- Add `last_seeded_at` timestamp to `artists` table
- First seed: fetch all 50, set `last_seeded_at = now()`
- Subsequent runs: use `publishedAfter = last_seeded_at - 1 day` → only genuinely new videos
- Deceased artists: once seeded, skip on daily cron entirely (they're not uploading anything new) — refresh monthly at most
- **Impact:** Daily cron drops from ~6,600 units → ~200–300 units

---

## 2. Home Page Quality

**Problem:** Home page was showing NULL view_count videos first (PostgreSQL NULLS FIRST default), and was also filtered to `book_recommended` artists only — hiding popular videos.

**Fixed today:**
- Added `nullsFirst: false` to `ORDER BY view_count DESC`
- Removed `book_recommended` filter from anonymous home page query

**Remaining ideas:**
- Enforce artist diversity: max 2 videos per artist in top 20 (Ranjani & Gayatri currently dominate)
- Add a freshness signal: `score = view_count × (1 / ln(days_since_published + 2))` to surface recently popular videos
- Three home page sections: "Featured" (manually curated), "Trending This Week" (algorithmic), "Legends" (deceased artists)
- For logged-in users: add "Because you watched X" label on recommendations
- Mark off-topic videos (`is_visible = false`) — e.g. Bollywood covers, film songs that crept in via high view count

## 3. Artist Page — Own Channel First + Sort/Filter

**Problem:** All videos for an artist are mixed together with no signal about source or quality.

**Ideas:**
- Show videos from the artist's own YouTube channel (`videos.channel_id = artists.youtube_channel_id`) first, labelled "Official"
- Videos from sabha/label channels shown second, labelled with channel name (e.g. "via Music Academy")
- Filter tabs at the top of the artist video list: **Most Viewed** | **Newest** | **Oldest**
- Show total video count on the artist card in the Browse page (e.g. "T.M. Krishna · 47 videos")
- Data already exists (`channel_id` on videos, `youtube_channel_id` on artists) — purely a query + UI change

---

## 4. User Mood Voting on Raga Chips

**Problem:** Mood → raga mappings are hardcoded in the search function. No community input.

**Ideas:**
- New DB table `raga_mood_votes`: `(user_id, raga, mood, vote: +1/-1)`
- When a user searches by mood (e.g. "peaceful") and sees results → show a small prompt: "Were these results right?" → 👍 / 👎
- On future raga detail page: show current mood tags with up/down voting
- Search function reads community vote aggregates to re-weight mood→raga associations
- Only trust votes with ≥10 total (prevents gaming)
- Hardcoded map acts as seed; community votes refine it over time
